# Continuous Improvement Workflow (Mandatory)

## Rule
Each sub-agent must submit at least **1 improvement proposal per cycle**.
Proposal must be approved by **IT Director (Astra)** before adoption.

## Improvement Proposal Template
- Role:
- Date/Time (WIB):
- Problem Statement:
- Root Cause Hypothesis:
- Proposed Improvement:
- Expected Impact (quality metric):
- Verification Method:
- Risk/Trade-off:
- Rollback Plan:

## Approval States
- `APPROVED` (implement this cycle)
- `APPROVED_WITH_CHANGES` (revise then implement)
- `REJECTED` (provide reason + alternative)
- `DEFERRED` (reconsider next assessment)

## Director Approval Criteria
- Improves reliability, quality, or test confidence
- Measurable outcome
- Low operational risk or has mitigation
- Aligns with strategic priorities

## Evidence Requirement
No improvement can be marked done without:
1. Before/after evidence
2. Test output or quality metric proof
3. Documentation update
